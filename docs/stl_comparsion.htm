More Details

1. std::vector

  • Stores elements contiguously like a C-style array.
  • Excellent cache performance.
  • Resizes dynamically, but may need to reallocate (which copies elements).
  • Supports random access via operator[] or at().

2. std::list

  • Doubly linked list.
  • Nodes stored non-contiguously; each element points to next and previous.
  • Insertions and deletions anywhere in the list are efficient (constant time).
  • No random access; iterating to an element takes O(n).
  • More memory overhead due to storing pointers.

3. std::array

  • Fixed-size array with size known at compile time.
  • Wrapper around a raw array but behaves like an STL container.
  • No dynamic memory allocation.
  • Supports random access in O(1).
  • Useful for small fixed-size sequences or as members of classes.

4. std::queue

  • Adapter container implementing FIFO.
  • Internally uses a container like std::deque by default (which allows efficient push/pop at both ends).
  • Exposes only limited interface: push(), pop(), front(), back().
  • No random access.

5. std::stack

  • Adapter container implementing LIFO stack.
  • Internally uses a container like std::deque by default (which allows efficient push/pop at one end).
  • Exposes only limited interface: push(), pop(), top().
  • No random access.

Summary of When to Use

Scenario Recommended Container
Need fast random access & dynamic resizing std::vector
Frequent insertion/deletion in middle std::list
Fixed size known at compile time std::array
Need FIFO queue std::queue
Need LIFO stack std::stack
